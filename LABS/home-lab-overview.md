# Home Lab Overview

## Purpose

My home lab is used to practice cybersecurity skills in a controlled, authorized environment. It gives me a place to build hands-on experience with Linux administration, system hardening, network scanning, scripting, and documentation without working on unauthorized systems.

## Lab Environment

My lab is focused on safe, repeatable practice and technical growth. I use it to test commands, document workflows, and improve my understanding of security concepts through direct hands-on work.

## Primary Uses

I use my lab to practice:

- Kali Linux setup and maintenance
- Linux terminal and Bash workflows
- Basic system hardening
- Network discovery and service enumeration
- Security-focused scripting
- Technical note-taking and project documentation

## Tools and Platforms

My lab work may include tools and platforms such as:

- Kali Linux
- Virtual machines
- Git and GitHub
- Bash
- Nmap
- Standard Linux administration utilities

## Operating Principles

I use this lab with the following principles:

- authorized systems only
- educational and defensive use only
- sanitized documentation
- repeatable workflows
- continuous improvement

## Why This Matters

Building a home lab helps me move beyond theory and practice cybersecurity skills in a structured way. It also helps me document my progress, strengthen problem-solving habits, and create a professional portfolio that reflects my development.

## Ongoing Growth

This lab will continue to evolve as I add more projects, improve my documentation, and expand the range of skills I am practicing.
