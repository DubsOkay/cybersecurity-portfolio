# Tools Used

## Purpose

This document lists the tools and platforms I use in my cybersecurity lab work and portfolio projects. My goal is not just to use tools, but to understand when to use them, how to document them clearly, and how to apply them responsibly in authorized environments.

## Core Tools

### Kali Linux
Used as a learning platform for Linux administration, security tooling, terminal workflows, and lab-based security practice.

### Bash
Used to build simple automation scripts, improve command-line confidence, and create repeatable workflows for administrative and security-related tasks.

### Nmap
Used in authorized lab environments for host discovery, port scanning, and basic service enumeration.

### Git
Used for version control, project tracking, and maintaining a clean history of improvements across documentation and scripts.

### GitHub
Used to publish projects, organize documentation, and present my cybersecurity learning journey in a professional format.

## Supporting Tools

### Linux System Utilities
Standard tools such as package managers, file permission commands, process tools, networking commands, and log utilities are used regularly for system administration and lab work.

### Virtual Machines
Used to create isolated and controlled environments for practice, testing, and documentation.

### Markdown
Used for structured technical writing, project documentation, notes, and writeups across this portfolio.

## How I Use These Tools

I use these tools to support skill development in:

- Linux administration
- system hardening
- network scanning
- scripting and automation
- technical documentation
- repeatable lab workflows

## Professional Approach

All tools in this portfolio are used for authorized lab, educational, and defensive security purposes only. I aim to use each tool with clear scope, documentation, and responsible practice.
